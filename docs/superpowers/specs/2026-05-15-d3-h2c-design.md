# D.3 — H2C (Cleartext HTTP/2) Design

**Date:** 2026-05-15  
**Status:** Approved  
**Goal:** Add `conn.PlaintextDialer` for H2C prior-knowledge connections (RFC 7540 §3.4) and `ClientOptions.DefaultScheme` so callers can set `:scheme: http` once at construction instead of per request.

---

## Background

All existing connections go through `TLSDialer`, which dials TCP, runs a TLS handshake, and asserts ALPN `"h2"`. `NewClientConn` is already transport-agnostic: it sends the HTTP/2 client connection preface (`PRI * HTTP/2.0\r\n\r\nSM\r\n\r\n`) and runs the SETTINGS exchange over whatever `net.Conn` is handed to it. H2C prior knowledge simply means connecting over plain TCP instead of TLS — the HTTP/2 layer is unchanged.

The `:scheme` pseudo-header currently defaults to `"https"` in `buildHeaders` when `req.Scheme` is empty. H2C targets expect `"http"`. Forcing per-request `req.Scheme = "http"` is boilerplate; a single `ClientOptions.DefaultScheme` field is cleaner.

---

## Architecture

```
caller
  └─ NewClient(ClientOptions{
         ConnOpts: {Dialer: &PlaintextDialer{}},
         DefaultScheme: "http",
     })
         │
         ▼
  PlaintextDialer.Dial  ──► net.DialContext("tcp", addr)  ──► net.Conn
         │
         ▼
  NewClientConn(ctx, net.Conn, opts)
         │  sends PRI * HTTP/2.0\r\n\r\nSM\r\n\r\n
         │  runs SETTINGS handshake
         ▼
  Conn  (identical to TLS path from here)
```

## Section 1: `conn.PlaintextDialer`

**File:** `conn/dial.go`

```go
// PlaintextDialer dials addr over TCP for H2C prior-knowledge connections
// (RFC 7540 §3.4). No TLS handshake or ALPN negotiation is performed.
// NewClientConn sends the HTTP/2 connection preface automatically.
type PlaintextDialer struct{}

// Dial dials addr over TCP and returns the raw connection.
func (d *PlaintextDialer) Dial(ctx context.Context, addr string) (net.Conn, error) {
    var nd net.Dialer
    return nd.DialContext(ctx, "tcp", addr)
}
```

No changes to `Dial` (the package-level function) or `NewClientConn`.

## Section 2: `ClientOptions.DefaultScheme`

**File:** `client/client.go`

Add to `ClientOptions`:

```go
// DefaultScheme is used as the :scheme pseudo-header when Request.Scheme
// is empty. Defaults to "https" when zero. Set to "http" for H2C targets.
DefaultScheme string
```

Store on `Client`:

```go
type Client struct {
    // ...existing fields...
    defaultScheme string // "https" when not set
}
```

In `NewClient` constructor, after existing option processing:

```go
c.defaultScheme = opts.DefaultScheme
if c.defaultScheme == "" {
    c.defaultScheme = "https"
}
```

In `buildHeaders`, replace the hard-coded `"https"` fallback:

```go
// Before:
scheme := req.Scheme
if scheme == "" {
    scheme = "https"
}

// After:
scheme := req.Scheme
if scheme == "" {
    scheme = c.defaultScheme
}
```

`buildHeaders` already receives `c.authority` as a parameter; thread `c.defaultScheme` the same way (add as parameter) **or** capture it in a closure — whichever matches existing style. Looking at the current signature `buildHeaders(req *Request, authority string)`, add `defaultScheme string` as a third parameter.

## Section 3: Error handling

- `PlaintextDialer.Dial` returns TCP errors verbatim (same pattern as `TLSDialer` for non-TLS dial errors).
- No `ErrALPNFailed` equivalent — there is no negotiation to fail.
- `DefaultScheme = ""` is zero-value safe; constructor normalises it to `"https"`.
- `req.Scheme != ""` always wins (per-request override, existing behaviour).
- Pool transport uses `ConnOptions.Dialer` transparently — H2C works with `TransportPool` without changes.
- Misconfiguration (plaintext dialer against TLS-only server) surfaces as `io.EOF` or a framing error from `NewClientConn`, same as any bad peer.

## Section 5: Non-goals

- HTTP/1.1 Upgrade (RFC 7540 §3.2) — not implemented.
- Auto-detecting scheme from dialer type at runtime — explicit `DefaultScheme` is cleaner.
- HTTPS → H2C redirect following.
- Unix socket or other transports — caller can implement `Dialer` interface directly.
